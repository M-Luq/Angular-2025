## ðŸš€ Day 18: Understanding Angular Directives

Angular Directives are essentially instructions for the DOM (Document Object Model). They tell Angular how to manipulate the structure or appearance of HTML elements. Think of them as **enhancements** or **superpowers** you can attach to your existing HTML tags.

### 1\. The Core Difference: Directives vs. Components

| Feature | Component | Directive | Analogy |
| :--- | :--- | :--- | :--- |
| **Decorator** | `@Component()` | **`@Directive()`** | The whole cake |
| **Template** | **Yes**, it has its own HTML. | **No**, they don't have a template. | The frosting, sprinkles, or filling |
| **Selector** | Element Selector (e.g., `<app-card>`) | **Attribute Selector** (e.g., `[appHighlight]`) | |
| **Purpose** | **Build** new UI blocks (like a button, a whole page section). | **Enhance** or **modify** existing HTML or Components. |

### 2\. Angular's Built-in Directives

Angular comes with several powerful directives, traditionally grouped into two types:

#### A. Structural Directives (Change the DOM Structure)

These directives literally add, remove, or manipulate elements in the DOM tree. They are always identified by an **asterisk (`*`) prefix** in the template.

| Directive | Type | What it Does | Example |
| :--- | :--- | :--- | :--- |
| **`*ngIf`** | Structural | Adds or removes an element based on a condition. | `<div *ngIf="isLoggedIn">...</div>` |
| **`*ngFor`** | Structural | Renders a list of elements based on a collection. | `<li *ngFor="let item of list">...</li>` |

> **ðŸ’¡ Modern Angular Note:** While `*ngIf` and `*ngFor` are the "classic" structural directives, the new **Control Flow Syntax** (`@if`, `@for`) is the recommended way to handle these structural changes in modern Angular versions. They achieve the same result but are simpler and more performant.

#### B. Attribute Directives (Change Appearance or Behavior)

These modify the appearance or behavior of an **existing** element.

| Directive | Type | What it Does | Example |
| :--- | :--- | :--- | :--- |
| **`ngModel`** | Attribute | Creates two-way data binding on form elements. | `<input [(ngModel)]="username">` |
| **`ngClass`** | Attribute | Dynamically adds or removes CSS classes. | `<div [ngClass]="{active: isActive}">...</div>` |

## 3\. Creating Your Own Directives: Why Use an Attribute Selector?

When you create a custom directive using the **`@Directive`** decorator, you **must** define a `selector`.

The main reason you should use an **attribute selector** (e.g., `selector: '[appHighlight]'`) for your custom directive is that directives are meant to be **superpowers applied to existing elements**, not new elements themselves.

| Selector Type | Example Code | Why We Use It |
| :--- | :--- | :--- |
| **Attribute Selector** (Directive) | `<button **[appTooltip]**>Info</button>` | It **decorates** or **enhances** the existing `<button>`. This is the directive's job\! |
| **Element Selector** (Component) | `<**app-tooltip**></app-tooltip>` | It **builds** a new UI element. This is the component's job. |

  * **Avoid Collisions:** Using a unique attribute name (like `[app-tooltip]`) prevents your code from conflicting with standard HTML tags (like `<div>`) or other libraries.

## 4\. Understanding Structural Directive Details (The Asterisk)

The asterisk (`*`) you see on `*ngIf` or `*ngFor` is **syntactic sugar** (a shorthand) for a much more complex underlying structure involving two key objects:

1.  **`TemplateRef`**: This is a reference to the block of HTML code **inside** the directive. It represents the *potential* content to be rendered.
      * **Injected via:** `private templateRef = inject(TemplateRef);`
2.  **`ViewContainerRef`**: This is a reference to the **place in the DOM** where the structural directive is used. It's the "parent container" where the content will be inserted or removed.
      * **Injected via:** `private viewContainerRef = inject(ViewContainerRef);`

When you write: `<div *ngIf="show">Content</div>`

The Angular compiler actually translates this into:

```html
<ng-template [ngIf]="show">
    <div>Content</div>
</ng-template>
```

  * **`ng-template`**: This is an Angular element that **is never rendered in the DOM**. It is a placeholder for content that Angular can insert later.
  * **The Power of No Asterisk:** You can use structural directives *without* the asterisk if you explicitly use the `<ng-template>` tag, giving you more direct control over when and where the content is rendered (which is what the `ViewContainerRef` is used for).

## 5\. Host Directives (A Modern Tool)

> **host directive why?**

**Host Directives** are a modern feature designed to **re-use logic across components**.

Imagine you have three different components (`<app-button>`, `<app-card>`, `<app-input>`) that all need the same behavior, such as a **Tooltip**.

Instead of copying and pasting the Tooltip directive code into the template of all three components, you can simply declare the Tooltip directive as a **Host Directive** on the components.

  * **What it does:** It automatically applies the logic of one directive (the **Host Directive**) to another component or directive (the **Host**).
  * **Goal:** Cleaner code, better reusability, and less boilerplate when multiple components need to share the same non-visual behavior or styling logic.
